Scroll Keeper -- Spacing Design Review

Three scroll-pacing approaches to reduce excess space between onboarding content sections

User Feedback

"Less space between content on scroll"

The original design used 133vh section heights with generous margins between content elements, creating excessive dead space that required too much scrolling to progress through the onboarding.

Identified Issues

  • Scroll sections at 133vh create ~33% excess vertical space per section
  • Hero section at 100vh with large margins feels disconnected from scroll content
  • Content element margins (1.5rem between verse/demo/heart and preceding text) add unnecessary gaps
  • Responsive breakpoints still too tall: 125vh at 1024px, 120vh at 768px
  • Total scroll distance for 3-section onboarding is ~5.5x viewport height -- could be ~3.5x

Changes Applied to 07-scroll-keeper.html (Direct Fix)

  • .scroll-section height: 133vh -> 110vh
  • .scroll-hero height: 100vh -> 85vh
  • .hero-bismillah margin-bottom: 1.5rem -> 1rem
  • .hero-scroll-arrow margin-top: 3rem -> 2rem
  • .sc-chapter margin-bottom: 0.5rem -> 0.35rem
  • .sc-initial margin-bottom: 0.3rem -> 0.2rem
  • .sc-subtitle margin-bottom: 1rem -> 0.7rem
  • .sc-rule margin-bottom: 1rem -> 0.7rem
  • .floating-verse margin-top: 1.5rem -> 1rem
  • .bilingual-demo margin-top: 1.5rem -> 1rem
  • .save-heart margin-top: 1.5rem -> 1rem
  • Responsive 1024px: 125vh -> 105vh
  • Responsive 768px: 120vh -> 100vh

All JavaScript animations remain untouched.

Option A: Compact (Direct Fix)

Open Preview

Design Approach

Uniform reduction across all sections. Every section shrunk by the same ratio, content margins tightened by ~30%. Straightforward and predictable.

Key Properties

  • All sections: 110vh (down from 133vh)
  • Hero: 85vh (down from 100vh)
  • Content margins reduced ~25-35% uniformly
  • Responsive breakpoints proportionally reduced

Best For

Preserving the original pacing feel while removing excess dead space. Safe, predictable change.

Option B: Progressive Density

Open Preview

Design Approach

First section stays spacious at the original 133vh, creating a grand entrance. Each subsequent section gets progressively tighter (115vh, 100vh), creating a "drawing in" effect that accelerates the user toward the finale. Content margins within each section also progressively tighten.

Key Properties

  • Section 1: 133vh (original -- grand, spacious)
  • Section 2: 115vh (moderate tightening)
  • Section 3: 100vh (most compact)
  • Hero: 100vh (full viewport for dramatic entrance)
  • Internal margins progressively shrink per section

Best For

Creating a narrative arc in the scroll pacing itself. The spacious opening gives the manuscript gravitas, then the accelerating rhythm draws the user in. Feels intentional rather than simply "shorter."

Option C: Snap Points

Open Preview

Design Approach

All sections reduced to 105vh with scroll-snap-type: y mandatory on the document. Each section snaps cleanly into the viewport as the user scrolls. The 5vh overshoot provides enough scroll travel for the JS animations to trigger, while the snap behavior ensures each "chapter" feels like turning a page.

Key Properties

  • All sections: 105vh with scroll-snap-align: start
  • Hero: 100vh with snap alignment
  • Finale: 100vh with snap alignment (up from 40vh)
  • Document: scroll-snap-type: y mandatory
  • Font sizes reduced slightly (0.82rem body text, 0.8rem subtitle)
  • Content margins tightened aggressively to fit within viewport

Best For

A "page-turning" manuscript feel. Each section occupies exactly one screen. The snap behavior prevents awkward half-section views. Most dramatic change in interaction model -- may feel jarring if scroll animations are important.

Recommendations

Best Overall: Option A (Compact) -- Directly addresses the feedback with minimal risk. Uniform reduction is predictable and preserves the animation timing feel.

Best Narrative Experience: Option B (Progressive Density) -- If the onboarding is meant to tell a story, the accelerating rhythm adds intentionality. The spacious opening gives weight to "Chapter I" while later sections feel more urgent.

Most Polished (if animations work well): Option C (Snap Points) -- The mandatory snap creates a true "manuscript page" experience. However, the animations depend on scroll travel within sections, so this needs careful testing. The 5vh overshoot may not provide enough travel for all animation phases to fully play out.
